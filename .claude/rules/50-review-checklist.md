# Review Checklist Rules

Before considering a task complete, verify:

## Structure
- Nuxt 4 structure is respected
- Files are placed in the correct directories
- Naming is consistent

## UI
- Layout is consistent
- Navigation works
- Empty states are not broken
- Status styling is consistent

## Code
- Imports are clean
- Types are clear
- No obvious dead code
- No broad `any`
- No needless duplication

## Server
- Mock routes return predictable data
- Shared types are reused when helpful
- Client/server boundaries are respected

## Checks
- lint
- typecheck
- tests if present
- build for meaningful changes

## Final response
Always summarize:
- files created
- files updated
- checks run
- remaining manual follow-up if any
